Sensory Analytics Engine

Project Overview

Our client, a global food-tech company, sought to streamline their operations by digitizing their daily activities, including sensory panel testing. Our team was tasked with developing custom software for the client, and my responsibility was to create their sensory test analysis within the application. Specifically, the client requested the implementation of the Two-Alternative Forced Choice (2-AFC) test, designed to assess significant differences between two samples.

Challenges

At the outset, I encountered a challenge: unfamiliarity with sensory analysis testing. Determined to deliver, I embarked on independent research to understand the mechanics of the 2-AFC test and its underlying logic. Translating this knowledge into Python code required building the algorithm from scratch. Luckily the client provided existing test results, so I used it as a trusted reference for my testing process.

Process

1. Brainstorming

This stage are about meet and discuss with the client about their expectation and how the algorithm should work in app.

2. Research

Conduct resarch about 2-AFC test.

3. Code

Code 2-AFC analysis in Python from scratch. Also, providing a data visualization for better understanding of the test results.

Result

I successfully developed the 2-AFC test in Python, seamlessly integrating it into the client's application. Importantly, the results generated by our digital implementation mirrored those of the manual test calculations provided by the client. This outcome validated the accuracy and reliability of our solution.

Data Stack

The project utilized a data stack consisting of Python, MongoDB and PostgreSQL.
